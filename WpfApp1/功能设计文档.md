## 1. 项目概述

### 1.1 产品定位

OpenP2P Launcher (OPL) 是一款基于 P2P 技术的网络联机工具，主要用于游戏联机、远程访问等场景。通过虚拟组网技术，实现多台计算机之间的直接连接，无需公网 IP 或端口映射。

### 1.2 目标用户

- 游戏玩家（需要联机玩游戏）
- 远程办公人员
- 技术人员（需要远程访问内网资源）
- 对网络配置不熟悉的普通用户

### 1.3 核心价值

- **简化配置**: 通过图形化界面简化 P2P 网络配置
- **一键联机**: 通过连接码实现快速联机
- **多种组网方式**: 支持 OpenP2P 和 EasyTier 两种组网方案
- **自动化管理**: 自动更新、自动启动等便捷功能

### 1.4 技术栈

- **前端框架**: WPF (Windows Presentation Foundation)
- **UI 库**: iNKORE.UI.WPF.Modern (Fluent Design)
- **后端语言**: C#
- **运行时**: .NET Framework 4.7.2
- **核心技术**:
  - OpenP2P (P2P 网络连接)
  - EasyTier (虚拟组网)

## 2. 核心功能模块

### 2.1 隧道管理模块

#### 2.1.1 功能概述

隧道管理是 OPL 的核心功能，负责创建、管理和监控 P2P 连接隧道。

#### 2.1.2 功能清单

| 功能      | 描述              | 优先级 |
| ------- | --------------- | --- |
| 创建隧道    | 手动创建新的 P2P 隧道连接 | P0  |
| 编辑隧道    | 修改已有隧道的配置参数     | P0  |
| 删除隧道    | 删除不需要的隧道配置      | P0  |
| 启用/禁用隧道 | 控制隧道的激活状态       | P0  |
| 隧道排序    | 调整隧道在列表中的显示顺序   | P1  |
| 快速添加    | 通过连接码一键添加隧道     | P0  |
| 导出连接码   | 将启用的隧道导出为连接码    | P0  |
| 隧道状态监控  | 实时显示隧道连接状态      | P0  |
| 复制连接地址  | 复制本地连接地址供应用使用   | P1  |
| 预设隧道    | 使用预定义的常用隧道配置    | P1  |

#### 2.1.3 隧道参数

**隧道配置参数**:

- **隧道名称** (AppName): 用户自定义的隧道标识
- **协议类型** (Protocol): TCP 或 UDP
- **目标 UID** (PeerNode): 远程设备的唯一标识符
- **远程端口** (DstPort): 远程设备的端口号
- **本地端口** (SrcPort): 本地映射的端口号
- **远程主机** (DstHost): 远程主机地址（默认 localhost）
- **启用状态** (Enabled): 1=启用, 0=禁用
- **白名单** (Whitelist): 访问控制白名单
- **中继节点** (RelayNode): 强制中继节点配置

#### 2.1.4 隧道状态

| 状态      | 颜色 | 说明          |
| ------- | -- | ----------- |
| 未启动/未启用 | 灰色 | 隧道未激活或未启用   |
| 连接中     | 橙色 | 正在建立 P2P 连接 |
| 连接成功    | 绿色 | P2P 连接已建立   |

#### 2.1.5 连接码格式

**标准格式**:

```
协议:UID:远程端口:本地端口
```

**简化格式** (默认 TCP，本地端口=远程端口):

```
UID:远程端口
```

**多连接格式** (使用分号分隔):

```
1:UID1:25565:25565;2:UID2:25566:25566
```

**示例**:

- `1:qwertyuiop:25565:25575` - TCP 协议，连接 UID 为 qwertyuiop 的设备，远程端口 25565，本地端口 25575
- `qwertyuiop:25565` - TCP 协议，连接 UID 为 qwertyuiop 的设备，远程和本地端口均为 25565

#### 2.1.6 业务规则

1. **端口冲突检测**: 同一本地端口不能同时被多个隧道使用
2. **自连接检测**: 禁止连接到自己的 UID
3. **运行限制**: 程序运行时禁止修改隧道配置
4. **频率限制**: 操作间隔至少 1 秒，防止频繁操作导致 BUG
5. **自动去重**: 添加相同 UID 和协议的隧道时，更新已有隧道而非创建新隧道

### 2.2 组网模块

#### 2.2.1 功能概述

提供两种虚拟组网方案，支持创建和加入虚拟局域网。

#### 2.2.2 OpenP2P 组网

**功能特性**:

- 创建虚拟网络（作为房主）
- 加入已有虚拟网络
- 自动分配虚拟 IP (10.0.23.x)
- 支持 TCP/UDP 协议

**操作流程**:

**创建网络**:

1. 设置通讯端口（默认 25674）
2. 点击"创建/开启网络"
3. 系统自动生成连接码并复制
4. 将连接码发送给其他参与者
5. 自动分配 IP: 10.0.23.1

**加入网络**:

1. 设置本机 IP 最后一位（2-20，不可重复）
2. 输入房主提供的连接码
3. 点击"连接网络"
4. 自动分配 IP: 10.0.23.x

#### 2.2.3 EasyTier 组网

**功能特性**:

- 基于 EasyTier 的虚拟组网
- 支持自定义节点服务器
- 实时显示网络节点列表
- 支持多种传输协议（TCP/UDP/KCP/QUIC）

**操作流程**:

**创建网络**:

1. 配置节点地址（格式: `tcp://节点地址:端口`）
2. 点击"创建网络"
3. 系统使用当前 UID 作为网络名称
4. 等待 5-8 秒建立连接
5. 将 UID 发送给其他用户加入

**加入网络**:

1. 配置节点地址
2. 输入网络名称（房主 UID 或自定义）
3. 点击"加入网络"
4. 查看左侧节点列表确认连接

**节点状态显示**:

- **Local** (绿色): 本地节点
- **Remote** (蓝色): 远程节点
- 显示信息: 主机名、IP 地址、延迟、收发流量

### 2.3 进程管理模块

#### 2.3.1 功能概述

管理 OpenP2P 和 EasyTier 后台进程的启动、停止和状态监控。

#### 2.3.2 OpenP2P 进程管理

**启动流程**:

1. 检查程序文件是否存在
2. 根据选择的节点设置服务器配置
3. 启动 openp2p.exe 进程
4. 重定向标准输出和错误输出
5. 解析输出日志，更新隧道状态
6. 启动 TCP/UDP 保活机制

**停止流程**:

1. 取消输出读取
2. 终止进程
3. 停止保活机制
4. 清理状态信息
5. 更新 UI 状态

**异常处理**:

- **端口占用**: 自动更换端口并重试
- **进程崩溃**: 2 秒后自动重启
- **安全拦截**: 提示用户添加安全中心白名单
- **网络错误**: 记录日志并提示用户

#### 2.3.3 日志解析规则

| 日志关键字                                   | 处理动作            |
| --------------------------------------- | --------------- |
| `autorunApp start`                      | 标记程序启动成功，状态变为绿色 |
| `autorunApp end`                        | 标记程序离线，状态变为橙色   |
| `LISTEN ON PORT ... START`              | 标记隧道连接成功，状态变为绿色 |
| `LISTEN ON PORT ... END`                | 标记隧道断开，状态变为橙色   |
| `ERROR P2PNetwork login error`          | 提示网络错误，自动重启     |
| `Only one usage of each socket address` | 提示端口占用，自动更换端口   |
| `no such host`                          | 提示 DNS 或网络错误    |
| `peer offline`                          | 提示对方不在线         |
| `NAT type:2`                            | 提示对称 NAT，可能影响连接 |
| `login ok`                              | 显示实际 UID        |

#### 2.3.4 保活机制

**TCP 保活**:

- 定期发送心跳包
- 检测连接状态
- 自动重连机制

**UDP 保活**:

- 定期发送心跳包
- 维持 NAT 映射
- 防止连接超时

### 2.5 更新模块

#### 2.5.1 功能概述

实现应用程序和核心文件的自动更新机制。

#### 2.5.2 更新流程

**检查更新**:

1. 从服务器获取 preset.json 配置文件
2. 比较本地版本号与服务器版本号
3. 检查核心文件 MD5 哈希值
4. 判断是否需要更新

**下载更新**:

1. 从镜像服务器下载更新包
2. 支持主镜像和 Gitee 备用镜像
3. 后台下载，不阻塞主程序
4. 显示下载进度

**应用更新**:

1. 主程序更新: 程序退出时自动解压覆盖
2. openp2p 更新: 立即解压覆盖
3. 更新完成后清理临时文件

#### 2.5.3 镜像策略

**主镜像**: `https://file.gldhn.top/`
**备用镜像**: `https://gitee.com/guailoudou/urlfile/raw/main/`

**切换逻辑**:

- 主镜像请求失败时自动切换到备用镜像
- 超时或网络错误时自动重试（最多 3 次）
- 记录镜像切换日志

## 5. 业务流程

### 5.1 程序启动流程

```
1. 检查单实例运行
   ├─ 已存在实例 → 激活已有实例 → 退出
   └─ 首次运行 → 继续

2. 检查管理员权限
   ├─ 非管理员 → 请求管理员权限 → 重启
   └─ 管理员 → 继续

3. 初始化 UI 主题
   ├─ 读取用户主题设置
   └─ 应用主题
```

```csharp
public static ConnectionParseResult ParseConnections(string input)
{
    if (string.IsNullOrWhiteSpace(input))
    {
        return ConnectionParseResult.Fail("连接码不能为空");
    }

    var connections = new List<ConnectionInfo>();
    var parts = input.Split(';');
    
    for (int i = 0; i < parts.Length; i++)
    {
        var part = parts[i].Trim();
        if (string.IsNullOrEmpty(part)) continue;
        
        var components = part.Split(':');
        
        try
        {
            var conn = ParseSingleConnection(components);
            connections.Add(conn);
        }
        catch (Exception ex)
        {
            return ConnectionParseResult.Fail(
                $"第 {i + 1} 个连接解析失败: {ex.Message}\n" +
                $"连接码: {part}\n" +
                $"示例格式: 1:UID:25565:25565"
            );
        }
    }
    
    return ConnectionParseResult.Success(connections);
}
```

#### 7.5.2 进程管理

**当前问题**: 进程管理逻辑分散，难以维护。

**改进建议**:

```csharp
public class ProcessManager : IDisposable
{
    private Process _process;
    private readonly ProcessConfig _config;
    private readonly ILogger _logger;
    
    public event EventHandler<ProcessOutputEventArgs> OutputReceived;
    public event EventHandler<ProcessStateChangedEventArgs> StateChanged;
    
    public async Task StartAsync(ProcessConfig config)
    {
        _config = config;
        
        var startInfo = new ProcessStartInfo
        {
            FileName = config.ExecutablePath,
            Arguments = config.Arguments,
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false,
            CreateNoWindow = true
        };
        
        _process = new Process { StartInfo = startInfo };
        _process.OutputDataReceived += OnOutputReceived;
        _process.ErrorDataReceived += OnErrorReceived;
        _process.EnableRaisingEvents = true;
        _process.Exited += OnProcessExited;
        
        try
        {
            _process.Start();
            _process.BeginOutputReadLine();
            _process.BeginErrorReadLine();
            
            StateChanged?.Invoke(this, new ProcessStateChangedEventArgs(ProcessState.Running));
        }
        catch (Exception ex)
        {
            _logger.LogError($"启动进程失败: {ex.Message}");
            throw;
        }
    }
    
    public async Task StopAsync()
    {
        if (_process == null || _process.HasExited) return;
        
        try
        {
            _process.CancelOutputRead();
            _process.CancelErrorRead();
            _process.Kill();
            await _process.WaitForExitAsync();
        }
        catch (Exception ex)
        {
            _logger.LogError($"停止进程失败: {ex.Message}");
        }
    }
    
    public void Dispose()
    {
        _process?.Dispose();
    }
}
```

#### 7.5.3 配置管理

**当前问题**: 配置读写分散，缺乏统一管理。

**改进建议**:

```csharp
public class ConfigService
{
    private readonly string _configPath;
    private Config _config;
    private readonly object _lock = new object();
    
    public ConfigService(string configPath)
    {
        _configPath = configPath;
        Load();
    }
    
    public Config GetConfig()
    {
        lock (_lock)
        {
            return _config;
        }
    }
    
    public void UpdateConfig(Action<Config> updater)
    {
        lock (_lock)
        {
            updater(_config);
            Save();
        }
    }
    
    public void Load()
    {
        try
        {
            if (File.Exists(_configPath))
            {
                var json = File.ReadAllText(_configPath);
                _config = JsonConvert.DeserializeObject<Config>(json);
            }
            else
            {
                _config = CreateDefaultConfig();
                Save();
            }
        }
        catch (Exception ex)
        {
            Logger.LogError($"加载配置失败: {ex.Message}");
            _config = CreateDefaultConfig();
        }
    }
    
    public void Save()
    {
        try
        {
            var json = JsonConvert.SerializeObject(_config, Formatting.Indented);
            Directory.CreateDirectory(Path.GetDirectoryName(_configPath));
            File.WriteAllText(_configPath, json);
        }
        catch (Exception ex)
        {
            Logger.LogError($"保存配置失败: {ex.Message}");
        }
    }
}
```
